# 뱅카우 GNB 2차 — 정보구조 개편 프로젝트

> 작성일: 2026.06.12
> 담당: 프로덕트 디자이너 윤현우
> 목적: 앱 정보구조 전면 개편을 위한 기획 문서. Claude Code 프로토타이핑 참고용.

---

## 목적

투자자가 필요한 정보를 찾을 수 있게 하세요.

뱅카우 투자자는 신청부터 정산까지 2년의 여정을 함께합니다. 각 단계에서 필요한 정보가 명확한 구조 위에 있을 때, 투자자는 불안 없이 다음 투자를 결심합니다.

---

## 배경 및 문제 정의

### 문제 현상

투자자가 신청부터 정산까지 여정에서 필요한 정보를 제때 찾지 못하고 있다.

### 문제 원인

정보 구조 정리 없이 기능과 정보가 덧붙이듯 추가되어왔고, 사용자 중심으로 한 번도 재정리된 적이 없다.

### 문제 원인 근거

- 자산 관련 정보가 여러 화면에 분산되어 있다
- 상품 상태에 따라 동일 정보의 진입 경로가 달라진다
- 투자 상품 안에 포함된 개별 송아지가 동일한 위계로 노출되어 정보 우선순위가 불명확하다
- 투자 상품보다 낮은 레벨의 송아지 정보가 메인에 지나치게 많이 노출되고 있다

### 이번 프로젝트에서 해결할 문제 정의

정보 구조가 정리되지 않은 채 기능과 정보가 레벨 구분 없이 누적되어, 투자자가 각 단계에서 필요한 정보를 찾는 데 불필요한 탐색이 발생하고 있다.

## 페이지별 정의

### 내 투자

**페이지 핵심 목적**
내가 넣은 돈이 지금 어떤 상태이고, 언제 얼마로 돌아오는지 확인하는 곳

**사용자가 주로 뭘 하려고 들어올까**

| 우선순위 | 행동 | 이유 |
|---|---|---|
| 1 | 내 투자 상품 확인 | 어떤 상품들에 투자했고, 각 상품 정보 확인 |
| 2 | 내 상품 새 소식 확인 | 투자한 소가 어떤 상태인지, 사육 잘 되고 있는지 |
| 3 | 투자금 확인 | 총 얼마 들어 있고, 각각 얼마 들어갔는지 |
| 4 | 정산까지 남은 기간 확인 | 언제 정산되는지 |
| - | 예상 수익 / 현재 수익 확인 | 지금 기준으로 수익이 얼마나 날 것 같은지 |
| - | 상품별 상세 정보 다시 보기 | 농장, 송아지 수, 예상 기간, 투자 조건 재확인 |

**사용자 유형**
- 투자하지 않은 사용자
- 투자 신청 중 사용자 (체결 전)
- 투자 직후 사용자
- 사육 중 사용자 (초반 ~6개월)
- 사육 중 사용자 (중반 6개월~1년 반)
- 정산 임박한 사용자
- 투자 정산된 사용자
- 여러 상품 동시 투자 중인 사용자

**사용자 시나리오 (이상적)**

시나리오 1 — 투자 신청 직후 (당일~체결일 전)
- "내가 투자 신청한 거 맞지? 제대로 된 거 맞아?"
- 신청 완료 후 내 투자 들어오면 "B상품 투자 신청중" 상태가 투자중 상품과 명확히 구분되어 보임
- 언제 신청 결과가 나오는지 날짜가 보여 안심하고 앱을 닫음

시나리오 2 — 체결일 당일 (알림 받은 직후)
- "1주 체결됐다고 알림왔네 내가 몇 주 투자 했더라?"
- 체결 알림 탭해서 바로 해당 상품 상세의 주문내역으로 딥링크 진입
- 3주 신청해서 1주 체결됐고, 나머지 2주는 환불됐음을 인지
- 내 투자 페이지에서 방금 체결된 B상품 1주 투자중 확인
- 내 계좌에도 2주 환불 금액이 들어왔고 내역도 있음 바로 출금

시나리오 3 — 사육 중 (이상적)
- "오늘 내 소 근황 올라왔네"
- 습관적으로 앱 열었는데 농장에서 올라온 사진이나 업데이트가 있어
- 내 소 몸무게가 얼마나 늘었는지, 사육 일수가 며칠인지 보임
- 짧게 보고 만족하고 닫음

시나리오 4 — 정산 임박 및 당일
- "정산 예정이라고 알림이 왔네 언제 된다는 거지?"
- 알림 클릭하고 내 투자로 이동 B 상품이 정산 예정 상태가 보임
- "아 B 상품이 정산이었구나 언제 된다는거야"
- 아 오늘 내일 중에 정산되어서 입금되는구나
- 정산 완료 알림톡 확인
- 정산 결과 페이지에서 B 상품 수익률 확인 상세에서 어떤식으로 결과가 나왔는지 세세하게 확인함
- 내 투자의 계좌에서 정산금 출금하거나, 지금 상품에 그대로 재투자 진행

### 내 자산 (내 투자 하위)

**페이지 핵심 목적**
지금 내가 가진 돈의 전체 그림을 한눈에 파악하는 곳

**사용자가 주로 뭘 하려고 들어올까**

| 우선순위 | 행동 | 이유 |
|---|---|---|
| 1 | 총 자산 / 총 투자금 확인 | 계좌 잔액과 투자 중인 금액을 합쳐 내 돈이 지금 얼마인지 전체 파악 |
| 2 | 입금 / 출금 | 새 상품에 투자하려고 돈을 넣거나, 정산금을 내 계좌로 빼려고 |
| 3 | 계좌 잔액 확인 | 지금 당장 투자 가능한 잔액이 얼마인지 |
| - | 거래내역 확인 | 투자 신청 시 출금, 정산금 입금, 환불금 입금이 정상적으로 처리됐는지 |
| - | 포인트 잔액 확인 | 쇼핑에 쓸 수 있는 포인트가 얼마 남았는지 |
| - | 수익률 확인 | 지금까지 정산된 상품 기준으로 내가 얼마나 벌었는지 |

---

### 투자내역 (주문내역)

**페이지 핵심 목적**
내가 신청/체결/정산한 투자 상품의 이력을 시간 순으로 확인하는 곳

**사용자가 주로 뭘 하려고 들어올까**

| 우선순위 | 행동 | 이유 |
|---|---|---|
| 1 | 투자 처리 내역 | 각 단계가 정상적으로 처리됐는지 확인하려고 |
| 2 | 과거 투자 이력 확인 | 내가 몇 호 상품에 얼마나 언제 투자했는지 히스토리 |
| - | 투자 신청 취소 | 체결 전에 마음이 바뀌었을 때 |

**주문내역 vs 거래내역 차이**
- 투자내역 → 투자 상품 단위 이력 (7-1호 3C 신청 → 1C 체결)
- 거래내역 → 계좌 입출금 단위 이력 (60,000원 출금, 20,000원 환불 입금)

---

### 정산내역

**페이지 핵심 목적**
정산 완료된 투자 상품의 수익 결과를 모아보는 곳

**사용자가 주로 뭘 하려고 들어올까**

| 우선순위 | 행동 | 이유 |
|---|---|---|
| 1 | 정산 결과 확인 | 얼마 수익 났는지, 수익률이 몇 %인지 확인하려고 |
| 2 | 누적 수익 확인 | 지금까지 뱅카우로 총 얼마 벌었는지 한눈에 보려고 |
| - | 세후 수익 확인 | 세금 떼고 실제로 손에 쥔 금액이 얼마인지 |
| - | 과거 정산 상품 조회 | 몇 호 상품이 언제 정산됐는지 히스토리 |

## 핵심 설계 원칙

1. **2년 공백을 설계하라** — 사육 기간 동안 앱을 열 이유를 만들어야 재투자율이 올라간다
2. **정보 위계를 지켜라** — 투자 상품 > 개별 송아지 순서로 정보 레벨을 유지한다
3. **각 단계의 불안을 해소하라** — 유저가 불안을 느끼는 순간에 필요한 정보가 있어야 한다
4. **재투자 루프를 완성하라** — 정산 → 내 자산 잔액 증가 → 재투자 탐색이 자연스럽게 이어져야 한다
5. **탭바는 자주 쓰는 것만** — 접근 빈도 낮은 기능은 탭바에 올리지 않는다

| 가상계좌 | NH농협 발급 뱅카우 전용 입출금 계좌 |

| 사육 중 | 투자 완료 후 만기 전 상태 |
| 출하 | 한우를 경매에 내보내는 것 |

## 유저 데이터 (기준: 2026.04.28)

- 총 고유 회원: 8,593명
- 핵심 타겟: 30~40대 남성 (전체 52.74%)
- 재투자율: 38.1% (2개 이상 상품 투자)
- 가입 후 첫 투자 중앙값: 14일
- 가입 당일 투자: 41.8%
- **가입 후 2주가 골든타임** — 온보딩 넛지는 당일~2주 집중

### 투자자 페르소나 3유형
- 탐색형 (~51%): 소액, 관망, 이탈 위험 높음
- 안전추구형 (~20%): 중간 금액, 신뢰 중시
- 수익추구형 (~22%): 고액, 데이터 중시
